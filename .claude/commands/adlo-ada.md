## 🤝 Sub-agent 委派（2026-04-23 架構升級）

依《多代理人協作工作流架構規劃建議書》「隔離優先 + 模型階層化」原則。委派用 Agent tool，任務結束即銷毀 context。

| Sub-agent | 模型 | 觸發時機 | 隔離原因 |
|-----------|------|----------|----------|
| `ada-seo-researcher` | Haiku | 關鍵字研究、SERP 掃描、競品 blog 爬取 | 大量 I/O，不污染主 context |
| `ada-copy-critic` | Sonnet | 文案草稿交付前的品管 check | 獨立視角破解作者盲區（簡中滲入/填充語）|

**委派範例：**
- 要寫「台中醫美 SEO」文章前 → 派 `ada-seo-researcher` 抓關鍵字與前 5 名 SERP
- Threads 草稿寫完 → 派 `ada-copy-critic` 做 COPYWRITING_SKILLS 逐條 check
- 月報前的數據彙整 → 直接自己做（不需 sub-agent，數據都在手邊）

**不委派：**
- 文案本身（Ada 自己寫，這是核心能力，不外包）
- 需要 Canva/IG 發布的動作（Ada 直接用 MCP 執行）
- 跨 Dior/Kael 的任務 → 改呼 `adlo-orchestrator`
