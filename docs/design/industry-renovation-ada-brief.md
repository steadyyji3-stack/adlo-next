## 背景

行業別新增父類別 **「裝潢修繕」**，4 子項：**裝潢 / 裝修 / 安裝 / 維修**。

工具分兩類處理：

| 類型 | 工具 | 狀態 |
|---|---|---|
| **自由輸入（不需文案）** | 店名產生器、評論連結 | ✅ Kael 已完成（4 子項直接加進分組選單，AI／字串代入自動處理） |
| **模板型（需 Ada 文案）** | 貼文產生器、我的這週、首頁試一下、LINE 推播 | ⏳ **等本 brief 文案** |

> 工程設計：模板型工具的 4 子項在 UI 顯示為 4 個選項，但**內部共用 1 組「裝潢修繕」模板**（不需各寫 4 份）。所以 Ada 只需產 **1 套** 裝潢修繕文案。

---

## 需要 Ada 產出的內容（共 1 套，3 個區塊）

### 區塊 A — GBP 貼文模板（餵 `src/lib/gbp-post-writer.ts`）
比照現有 `餐飲` 結構，提供 **裝潢修繕** 的 7 天貼文骨架。每則：
```jsonc
{ "day": "週一", "category": "案例展示|施工須知|報價透明|...", "title": "≤20字", "content": "≤120字，含 1 個 CTA" }
```
參考現有 `餐飲: [...]`（gbp-post-writer.ts:103）的語氣與長度。裝潢修繕適合的主題池：完工前後對比、施工流程透明、報價說明、保固、注意事項、季節提醒、客戶見證。

### 區塊 B — 行業標籤（餵 `INDUSTRY_TAGS`，line-broadcast.ts）
3 組各 3–5 個標籤（對應現有 3 組結構）：例如「服務項目 / 信任賣點 / 行動呼籲」。供使用者勾選帶入貼文。

### 區塊 C — LINE 推播模板（餵 `line-broadcast.ts` 兩個字典，line 66 & 298）
比照 `餐飲` 提供裝潢修繕的 LINE OA 推播初稿（含主旨 + 內文 + CTA）。注意 LINE 語氣較私訊、可帶限時/預約。
